# Функционалности (MK Company Data)

Овој проект е Laravel апликација која:
- влече (scrape) листи на компании од јавниот регистар `zk.mk`,
- ги зачувува локално (во база),
- овозможува работа преку админ панел (`/admin`) за компании и понуди,
- по потреба генерира понуди со AI (структуриран излез: `title` + `content`),
- има Artisan команди за масовно збогатување со е-пошти.

## Што има апликацијата (накратко)

- **Собирање компании (scraping)** од `zk.mk` по сектори + пагинација
- **Компании (Filament /admin)**: листа, пребарување, филтри, уредување
- **Email enrichment** (конзолна команда) + **рачно додавање email**
- **Activity index** (`activity_index`) + бројач `scrape_count` (за рангирање/аналитика)
- **Понуди (Filament /admin)**: AI генерирање содржина за понуда + избор на компанија
- **AI препорака на компании** за понуда + **target list** (зачувано во `offer_targets`) + UI таб за менаџирање
- **Статистика (Table II)** преку команда `app:company-statistics`
- **Детекција на проблематични компании (AI data quality)**: команда `app:detect-invalid-companies` + badge/филтер во листата на компании

## Последни додатоци

- **`scrape_count` на компании**: нова колона која брои колку пати истата компанија (по име) е „погодена“ низ повеќе скрејпинг извршувања.
- **Автоматска пресметка на `activity_index`**: индекс (0–1) што се пресметува по формула \(AI = 0.5E + 0.3P + 0.2F\), каде:
  - **E** = 1 ако има валиден email, инаку 0
  - **P** = 1 ако има и телефон и адреса, инаку 0
  - **F** = нормализиран `scrape_count` (min-max 0–1 преку сите компании)
- **Нова команда**: `php artisan app:calculate-activity-index` — пресметува за сите компании и печати min/max/average.
- **Автоматско повикување по скрејпинг**: по `php artisan app:scrape-companies-command` автоматски се повикува пресметката на `activity_index`.
- **Нова команда (статистика / Table II)**: `php artisan app:company-statistics` — печати табела со метрики (вкупно компании, број сектори, покриеност со email, “duplicate” стапка од `scrape_count`, итн.).
  - **Вредности што не се во база** (на пр. *Initial Email Coverage %* и *Average Scraping Time per Sector*) се внесуваат во `config/company_statistics.php`.
- **AI препорака на компании при креирање понуда**: во креирање на понуда има секција **“AI Recommend Companies”** која, според наслов/опис/сектор, предлага компании (од топ 50 по `activity_index` за сектор) со кратка причина. Можеш да означиш (checkbox) и избраните ќе се зачуваат како **target list** за понудата.
- **UI за Target компании во понуда**: во **Понуди → Детали** има таб **„Target компании“** каде ги гледаш избраните компании, можеш да додадеш нова (рачно), да уредиш причина или да избришеш таргет.
- **AI проверка на квалитет на компании**: `php artisan app:detect-invalid-companies` означува компании како `duplicate|inactive|inconsistent` и во **Компании** се прикажува badge „Квалитет“ + филтер **„Само означени“**.

## Како да тестирам (брзо)

1) **Миграции (нови табели/колони)**

```bash
php artisan migrate
```

2) **`scrape_count` + `activity_index`**
- Пушти скрејпинг:

```bash
php artisan app:scrape-companies-command
```

- На крај автоматски ќе се повика пресметката на индексот. Ако сакаш рачно:

```bash
php artisan app:calculate-activity-index
```

Забелешка: ако користиш SQLite и имаш пуштено паралелно `php artisan serve` / queue / други процеси што ја користат базата, можеш да налеташ на lock. Во тој случај, пушти ја командата повторно или привремено стопирај ги другите процеси.

3) **Статистика (Table II)**

```bash
php artisan app:company-statistics
```

Параметрите што не се следат во база ги подесуваш во `config/company_statistics.php`.

4) **AI Recommend Companies + target list**
- Во `.env` постави `OPENAI_API_KEY`.
- Влези во `/admin` → **Понуди** → **Нова понуда**.
- Пополни:
  - „Наслов (за AI препораки)“
  - „Дополнителни информации“
  - „Сектор (за AI препораки)“
- Кликни **“AI Recommend Companies”** → ќе се прикаже табела со препораки и checkbox.
- Означи компании и креирај понуда → избраните се зачувуваат во табелата **`offer_targets`** (target list) со причина.

5) **AI детекција на проблематични компании (data quality)**
- Во `.env` постави `OPENAI_API_KEY`.
- Пушти:

```bash
php artisan app:detect-invalid-companies
```

- Во `/admin` → **Компании** ќе видиш:
  - badge колона **„Квалитет“**
  - филтер **„Само означени“**

## 1) Скрејпинг на компании од `zk.mk`

- **Што прави**: За дефинирани сектори, ги чита резултатите од `https://zk.mk/<сектор>` и ги снима/ажурира во табелата `companies`.
- **Како работи**:
  - Пагинацијата е автоматска: додека има `next` линк, продолжува да вчитува наредни страници.
  - За секој резултат извлекува: **име**, **град**, **адреса**, **телефон** (и пробува да најде **email** во HTML).
  - Во базата користи `updateOrCreate` по `name` (ако веќе постои компанија со исто име, ја ажурира).
- **Команда**:

```bash
php artisan app:scrape-companies-command
```

- **Сектори** (пример): градежни, ИТ, здравство, туризам, банки, општини, образование, авиокомпании, осигурување, финансии (види `App\Enums\SectorEnum`).

## 2) Компании (админ панел)

Во `/admin` има ресурс **„Компании“** со:
- **Листа** со пребарување и колони: име, сектор, град, адреса, е-пошта, телефон, `activity_index`.
- **Филтри**: по сектор и по град (градовите се полнат динамички од базата).
- **Креирање/уредување**: истите полиња преку форма.
- **`activity_index`**: поле во `companies` (default 0). Во UI е прикажано и е „disabled“ во формата (моментално нема логика што го пресметува автоматски).

## 3) Збогатување (enrichment) на е-пошти

- **Што прави**: Обработува компании и пробува да најде е-пошта со повеќе стратегии.
- **Стратегии (редослед)**:
  1. **Вебсајт скрејпинг**: генерира можни домени од името на компанијата, пробува `http/https` и `www`, па извлекува email од HTML; дополнително пробува и „contact/about“ патеки (`/contact`, `/kontakt`, `/za-nas`, итн.).
  2. **Google пребарување**: прави request кон Google резултати и бара email во HTML (може да не работи стабилно поради rate limiting/бот заштита).
  3. **Патерни**: ако не најде ништо, генерира „најчест“ email како `info@<domain>` (со приоритет на `.mk` и `.com.mk`).
- **Команда**:

```bash
php artisan app:enrich-company-emails --limit=100
```

Опции:
- `--all` обработува „сите“ (без лимит).
- `--skip-existing` (стандардно true) ги прескокнува компаниите што веќе имаат email.
- `--detailed` печати детално за секоја компанија (OK/SKIP/FAIL).

## 4) Рачно додавање е-пошта на компанија

Ако сакате рачно да додадете/замените email за компанија (со partial match по име):

```bash
php artisan app:add-company-email "Име" "info@domain.mk"
```

## 5) Понуди (админ панел) + AI генерирање

Во `/admin` има ресурс **„Понуди“** со:
- **Креирање нова понуда**: избирате компанија и внесувате „Дополнителни информации“.
- **AI генерирање**:
  - При креирање, системот гради prompt од избраната компанија + внесените информации.
  - Потоа се повикува AI агент (`App\Ai\Agents\OfferAgent`) кој враќа **структуриран JSON** со `title` и `content`.
  - Овие вредности се зачувуваат во табелата `offers` (заедно со `company_id`).
- **Преглед/уредување**: постои „Преглед“ (View) и „Уреди“ (Edit) за понудите.

Важно:
- За AI да работи, во `.env` треба да има **`OPENAI_API_KEY`** (види `config/ai.php` и `config/openai.php`).

## 6) Контролна табла (Dashboard)

Постои „Контролна табла“ во админ панел. Има и widget `StatsOverview` што пресметува:
- вкупно компании
- компании со поставен email

## 7) Брзо стартување (за работа со функционалностите)

- **Инсталација**:

```bash
composer run setup
```

- **Локален развој (server + queue + vite)**:

```bash
composer run dev
```

- **Админ корисник** (за да се логирате на `/admin`):

```bash
php artisan make:filament-user
```
